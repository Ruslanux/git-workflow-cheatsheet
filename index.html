git commit vs git push

git commit

  • Назначение Сохраняет снимок изменений в локальную историю Git
  • Область Local Изменения остаются только на вашей машине
  • Требует Выполнения git add (индексация изменений)
  • Совместная работа Другие разработчики не видят ваших коммитов
  • Аналогия Как сохранение файла (Ctrl+S)

git push

  • Назначение Передает локальные коммиты на удаленный сервер
  • Область Remote Обновляет репозиторий на сервере
  • Требует Хотя бы один локальный коммит для передачи
  • Совместная работа Делает работу доступной для команды
  • Аналогия Как загрузка файла в облако

Pull Request (PR)

Что такое Pull Request?

Pull Request — это запрос на слияние изменений из одной ветки в другую. Это основной механизм code review в командной разработке. PR позволяет обсудить изменения, оставить комментарии и убедиться в качестве кода перед слиянием.

Жизненный цикл PR

  • 1. Open PR создан, ожидает review
  • 2. Review Команда проверяет код, оставляет комментарии
  • 3. Changes Автор вносит правки по комментариям
  • 4. Approved Reviewer одобрил изменения
  • 5. Merged PR слит в основную ветку

Команды GitHub CLI (gh)

  • gh pr create — создать PR
  • gh pr list — список открытых PR
  • gh pr view — просмотр PR
  • gh pr checkout 123 — переключиться на PR
  • gh pr merge — слить PR
  • gh pr review --approve — одобрить PR

Лучшие практики для Pull Request

✓ Маленькие PR

200-400 строк кода. Легче ревьюить и меньше конфликтов.

✓ Понятное описание

Что изменено, зачем, как тестировать.

✓ Одна задача = один PR

Не смешивайте фичи и рефакторинг.

✓ Тесты проходят

CI должен быть зеленым перед merge.

Типичные сценарии работы

1. Сохранение изменений (базовый workflow)

1
Проверить статус изменений
git status
2
Добавить файлы в индекс
git add .
3
Создать коммит с сообщением
git commit -m "Описание изменений"
4
Отправить на сервер
git push origin main

2. Работа с feature-веткой

1
Создать и переключиться на новую ветку
git checkout -b feature/new-feature
2
Внести изменения и закоммитить
git add . && git commit -m "Add new feature"
3
Отправить ветку на сервер
git push -u origin feature/new-feature
4
Создать Pull Request на GitHub
gh pr create --title "New feature" --body "Description"

3. Получение изменений от команды

1
Получить и слить изменения
git pull origin main
2
Или получить без слияния (для проверки)
git fetch origin
3
Посмотреть что изменилось
git diff main origin/main

4. Отмена изменений

1
Отменить изменения в файле (до git add)
git checkout -- filename.rb
2
Убрать файл из индекса (после git add)
git reset HEAD filename.rb
3
Отменить последний коммит (сохранить изменения)
git reset --soft HEAD~1
4
Отменить коммит полностью (создать revert-коммит)
git revert HEAD
